操作系统要点

LazySwifts 2021-05-28 22:27:55 Operating System

# 概述

操作系统的特征：并发、共享、虚拟、异步

操作系统的功能：处理机管理、存储器管理、文件管理、设备管理

发展：手工、单道批处理（自动性、顺序性、单道性）、多道批处理（多道、宏观上并行、微观上串行）、分时、实时、网络和分布式

运行机制：内核态（特权指令）、用户态（仿管指令）

中断：

  • 内中断（异常）：非法操作、缺页等
  • 外中断：外设请求、仿管中断等

中断处理过程：关中断、保存断点、中断服务程序寻址；保存现场和屏蔽字、开中断、执行中断服务程序、关中断、恢复现场和屏蔽字、开中断、中断返回

体系结构：大内核和微内核

并发的特点：失去封闭性、不可再现性、间断性

并发和并行的区别

并行：多个事件在同一时刻发生

并发：多个事件在同一时间间隔内发生

# 进程管理

# 进程

概念：是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位，是程序的一次执行过程

特点：并发性、共享性、动态性、独立性、异步性、结构性

状态：运行态、就绪态、阻塞态、创建态、结束态

进程实体：程序段、相关数据段、PCB

PCB 是进程存在的唯一标志

一个进程可以对应几个程序，一个程序也可以对应几个进程

进程创建、撤销、切换等都要在内核态下完成

PCB 的组织：链接方式和索引方式

进程通信：

  • 共享存储：效率高，两次拷贝，配合信号量机制
  • 消息传递：消息缓冲队列，四次拷贝，无格式字节流；直接、间接两种
  • 管道通信：用 pipe 文件，四次拷贝（输入文件-->内存-->内核-管道/队列-->内存-->输出文件），半双工，数据是一次性读取

# 线程

概念：是一个基本的 CPU 执行单元，是程序执行流的最小单元，是独立调度和分派的基本单位，但不拥有资源

目的：提高资源利用率和系统吞吐量，减少程序在并发执行时付出的时空开销。线程共享空间，切换时减少缺页中断，切换开销也小；同一进程中的多个线程可以并发执行，提高系统的并发性；共享地址空间，同步和通信容易实现

不同的线程可以执行相同的程序，也有各种状态

线程切换时不一定进程切换，所以平均每次切换开销小

实现方式：用户级线程、内核级线程 —— 1:1; 1:n; n:m

为什么引进进程？

为了描述程序执行过程的动态特征并更好地支持和管理多道程序的并发执行

进程如何解决问题？

进程把能够使别程序运行态的一些变量存放在 PCB 中，通过这些变量系统能够更好地了解进程的状况，并在适当时进行进程切换，以避免一些资源的浪费，甚至划分更小的调度单位——线程来提高系统的并发度

# 处理机调度

层次：

  • 作业调度：从外存后备状态中的作业中挑选一个为其分配资源，建立相关进程（外存-->内存）
  • 内存调度：将暂时不运行的进程调至外存，将可以运行的调入内存（就绪-->挂起，挂起-->就绪）
  • 进程调度：在就绪队列中选择一个进程分配处理机（就绪-->运行）

过程：请求调度-->进程调度-->进程切换

不能进行进程调度和切换的情况：

  • 在处理中断的过程中
  • 进程在操作系统内核程序临界区中（但是可以进行内存调度、作业调度，通过中断实现）
  • 其他原子操作过程中

方式：剥夺式、非剥夺式

原则：CPU 利用率、系统吞吐量（单位时间内完成的作业数量）、周转时间、带权周转时间、等待时间、响应时间（交互式系统）

🔺调度算法：有的适合进程调度，有的适合作业调度，有的都适合

  • FCFS：不可剥夺，效率低，对长作业有利，有利于 CPU 繁忙型作业，不利于 I/O 繁忙型作业
  • SJF：对长作业不利（饥饿现象），平均等待时间和平均周转时间最短
  • 优先级
  • 高响应比优先：有利于短作业，克服了饥饿状态，兼顾了长短作业，计算响应比开销大
  • 多级反馈队列：1 ~ n-1 级队列使用 FCFS 算法，第 n 级使用时间片轮转算法，仅当第 1 级队列为空时，才调度第 2 级队列中的进程；在优先级越高的队列中，每个进程的运行时间片越小
    • 终端型作业用户：短作业优先
    • 短批处理作业用户：周转时间较短
    • 长批处理作业用户：会先经部分执行，不会长期得不到处理
  • 时间片轮转：剥夺式，兼顾长短作业，平均等待时间较长，上下文切换浪费时间，适用于分时系统

# 进程同步

临界资源：一次仅允许一个进程使用的资源

临界区：访问临界资源的那段代码

同步：直接制约关系，协调等待的制约关系（协作）

互斥：竞争使用临界资源的间接制约关系（竞争）

准则：

  • 空闲让进
  • 忙则等待
  • 有限等待：保证进程在有限时间内进入临界区
  • 让权等待：进程不能进入临界区时，立即释放处理器

互斥实现方法：

  • 软件：

    • 单标志法：某个进程不再进入临界区，另一个也进不去，违背空闲让进
    • 双标志法先检查：可能同时进入，违背忙则等待
    • 双标志法后检查：可能都进不去，导致饥饿现象
    • Peterson's Algorithm

  • 硬件：

    • 中断屏蔽方法

    • 硬件指令方法

      • TestAndSet
      • Swap

      缺点：进程等待进入临界区时要耗费处理机时间，不能实现让权等待。从等待进程中随机选择一个进入临界区，有的进程可能一直选不上，导致出现饥饿现象

信号量：

解决同步互斥问题，wait(S) 和 signal(S) 原语

P 操作：>=0，执行；<0，阻塞

V 操作：>0，继续执行；<=0，通知阻塞变为就绪

信号量在同步中一般设为 0，互斥中一般设为 1 或资源数目

  • 整型信号量，资源数目 S
  • 记录型信号量

管程：

信号量操作容易造成死锁，管程的特性保证了进程互斥，无需程序员自己实现互斥，降低死锁发生的可能，同时管程提供了条件变量，可以灵活实现进程同步。

管程定义了一个数据结构和能为并发进程所执行的一组操作，这组操作能同步进程和改变管程中的数据。

管程把对共享资源的操作封装起来，只有管程内的过程可以访问；每次仅允许一个进程进入管程，实现进程互斥

# 🔺死锁

定义：指多个进程因竞争资源而造成的一种僵局，若无外力作用，这些进程都将无法向前推进

原因：

  • 系统资源的竞争
  • 进程推进顺序非法

必要条件：

  • 互斥
  • 不剥夺
  • 请求保持
  • 循环等待

死锁预防：破坏必要条件之一，进程初始化时间被延长

  • 破坏互斥
  • 一个进程已占有的资源可以被剥夺
  • 预先静态分配，进程在运行前一次申请完它所需要的全部资源——容易导致饥饿现象、系统资源被浪费
  • 顺序资源分配，为资源编号，只要进程提出申请分配资源 ，则该进程在以后的资源申请中就只能申请编号大于 的资源

死锁避免：执行过程中，银行家算法——避免系统进入不安全状态，需要知道将来的资源需求

  • 矩阵

死锁检测：通过剥夺解除死锁，造成损失，资源分配图

  • 死锁定理：S 为死锁的条件当且仅当 S 状态的资源分配图是不可完全简化的

死锁解除：资源剥夺法、撤销进程法、进程回退法（让进程资源释放资源）

饥饿和死锁的区别

  • 饥饿的进程可以只有一个，死锁的进程有多个
  • 饥饿的进程可以是一个就绪进程，而死锁的进程必定阻塞

作业和进程的关系

  • 进程是系统资源的使用者，用户要求计算机完成的任务是作业

# 内存管理

# 基本内存管理

概念：操作系统对内存的划分和动态分配

功能：内存空间的分配和回收、地址转换（多道）、内存空间的扩充（虚拟存储、自动覆盖）、存储保护

🔺程序装入和链接：

  • 编译，源代码-->目标模块

  • 链接，目标模块 + 库函数-->装入模块，形成逻辑地址

    • 静态链接，运行之前
    • 装入时动态链接，装入内存时
    • 运行时动态链接，在程序执行中需要该目标模块时才进行的，便于修改和更新、共享

    段式管理方便动态链接

  • 装入，将装入模块装入内存

    • 绝对装入（单道）

    • 可重定位装入（静态），地址变换在装入时一次完成

      必须给作业分配全部内存空间，运行期间作业不能在内存中移动、申请内存空间

    • 动态运行时装入（动态），程序在内存中发生移动时需要，装入的是相对地址，需要重定位寄存器

      可以将程序分配到不连续的存储区中；在程序运行之前可以只装入部分代码；在运行期间可以动态申请分配内存；便于程序段的共享，可以向用户提供一个比存储空间大得多的地址空间

内存保护：上下限寄存器、重定位寄存器、界地址寄存器

覆盖：将用户空间分为一个固定区和多个覆盖区，同一进程可以部分装入、逐步覆盖，用于多道

交换：不同进程之间交换，换进换出内存

🔺连续分配方式：

  • 单一连续分配：无外部碎片，只能用于单用户单任务，有内部碎片，存储器利用率低
  • 固定分区分配：分区说明表，主存利用率低，存在内部碎片
  • 动态分区分配：存在外部碎片，通过紧凑技术解决，依靠动态重定位寄存器，费时
    • 首次适应算法，最好最快，出现小的空闲分区
    • 最佳适应算法，性能很差，产生最多的外部碎片
    • 最坏适应算法，会导致没有可用的大内存块，性能差
    • 临近适应算法，导致在内存的末尾分配空间分裂成小碎片

🔺非连续分配方式：

  • 基本分页存储：将主存空间划分为大小相等且固定的块，作为主存的基本单位，每个进程也以块为单位进行划分、申请空间

    不会产生外部碎片，但是会产生页内碎片，两次访存

    页表结构：页号，块号

    地址结构：页号，页内偏移量（页大小固定，一维）

    块表：相联存储器（TLB），一次访存

  • 基本分段存储：按照自然段划分逻辑空间

    不会产生内部碎片，可能产生外部碎片，两次访存

    段表结构：段号，段长，本段在主存中的始址

    地址结构：段号，段内偏移量（段内连续，段间可以不连续，二维）

  • 段页式存储：先将地址空间分为逻辑段，每段分成固定大小的页

    地址结构：段号、页号、页内偏移量

    在一个进程中，段表只有一张，页表有多张，三次访存

    段表结构：段号，页表长度/段长，页表起始地址

# 虚拟内存管理

将程序的一部分装入内存，另一部分驻留在外存等待调入。

局部性原理：时间局部性（循环操作）、空间局部性（顺序存放）；利用局部性原理可以实现高速缓存

特征：多次性、对换性、虚拟性（逻辑上）

实现方式：请求分页、请求分段、请求段页式

硬件要求：内存外存、页表段表、中断机构、地址变换机构

🔺页面置换算法：

  • 最佳（OPT）置换算法：淘汰最长时间内不会被访问的页面，理想算法，用于评价其他算法
  • 先进先出（FIFP）页面置换算法：队列类，实现简单，性能差，Belady现象（物理块数增加缺页数却增加）
  • 最近最久未使用（LRU）置换算法：堆栈类，实现困难，开销大
  • 时钟（CLOCK）算法/最近未用（NRU）算法：使用位 0/1
  • 改进型 CLOCK 置换算法：修改位、使用位

驻留集：给一个进程分配的物理页框的集合

工作集：某段时间间隔内，进程要访问的页面集合

一般驻留集要大于工作集

页面分配策略：

  • 固定分配局部置换
  • 可变分配全局置换
  • 可变分配局部置换

抖动：频繁的页面调度行为，原因是某个进程频繁访问的页面数量高于可用的物理页帧数量

虚拟内存容量要满足的条件：

实际容量 <= 内存容量 + 外村容量

最大容量 <= 计算机的地址位数所能容纳的最大容量

# 文件管理

文件概念：以计算机硬盘为载体的存储在计算机上的信息集合

属性：名称、标识符、类型、位置、大小、保护、时间等，文件信息保存在目录结构中，目录结构保存在外村中

基本操作：创建、读、写、重定位、删除、截断

文件的打开与关闭：

  • 首次使用文件时，系统调用 open 将文件属性复制到内存打开文件表中的一个表目中，返回表目索引（指针）给用户；之后通过该索引对文件进行操作。单个进程也有自己的打开文件表。
  • 删除时，将文件打开计数器 -1，并且删除在该进程打开文件表中的表项，若计数器此时为 0，则系统回收分配给文件的内存等资源，删除系统打开文件表中的相应条目并且释放文件控制块

# 逻辑结构

无结构文件（流式文件）：以字节为单位，适合字符流的文件（源程序文件、目标代码文件）

🔺有结构文件（记录式文件）：

  • 顺序文件：批量操作时效率最高，可以存储在磁带上，对增删改查比较困难；串结构、顺序结构

  • 索引文件：定长记录文件可以通过计算得到地址；变长则需要顺序遍历

    索引表可以加入检索速度，索引表本身是定长记录的顺序文件

  • 索引顺序文件：将顺序文件中的记录分为若干组，为其建立一张索引表，索引表中的索引项为每组的第一条记录的逻辑地址

  • 直接文件或散列文件：直接计算出物理地址

目录结构：按名存取，目录中存放文件名和索引结点编号，文件被打开时，磁盘索引结点复制到内存的索引结点中

  • 单级目录，目录中存放 FCB
  • 两级目录，将文件目录分成主文件目录（MFD）和用户文件目录（UFD），解决命名冲突
  • 多级目录，层次结构清晰
  • 无环图目录，便于文件共享

🔺文件共享：

  • 基于索引结点的共享方式（硬链接）：文件目录中设置相应索引结点的地址，在索引结点中还有链接计数 count

  • 基于符号链实现文件共享（软链接）：创建 LINK 类型的新文件，其中包含被链接文件 F 的路径名，只有文件的拥有者才拥有指向其索引结点的指针。由于根据路径名读取，需要多次读盘，符号链的索引结点也要耗费一定的磁盘空间

    优点：网络共享方便，只需要提供文件路径

文件保护：口令、加密、访问控制（访问控制列表 ACL）

# 物理结构

文件系统层次：用户调用接口、文件目录系统、存取控制验证模块、逻辑文件系统与文件信息缓冲区、物理文件系统、辅助分配模块、设备管理程序模块

目录实现：线性列表、哈希表

🔺文件实现：

  • 连续分配：每个文件占有一组连续的块，寻道数和寻道时间最小

    优点：实现简单、存取速度快，支持顺序访问和直接访问，访盘 1 次

    缺点：文件长度不宜动态增加，反复增删文件后会产生外部碎片，只适用于长度固定的文件

  • 链接分配：离散分配，消除了外部碎片，访盘 n 次

    优点：顺序访问，提高磁盘空间利用率，增删改方便

    缺点：不能随机访问

    分类：

    • 隐式链接，目录包括文件的第一块指针和最后一块指针，只能通过指针顺序访问文件，并且指针占用空间
    • 显式链接，文件分配表（FAT），记录文件各块之间的先后链接关系，还标记了空闲的磁盘块

  • 索引分配：把每个文件的所有盘块号都集中放在一起构成索引块（表）

    目录中包含文件的索引块地址，通过索引块可以找到每一逻辑块对应的物理块

    优点：支持直接访问，没有外部碎片，易于增删改

    缺点：索引块增加系统存储空间的开销

    索引块组织方式：

    • 链接方案
    • 多层索引，访盘 m + 1 次
    • 混合索引

🔺文件存储空间管理：

  • 空闲表法：连续分配方式

    表结构：序号、第一个空闲盘块号、空闲盘块数

    和动态分区分配类似，也采用首次适应算法、循环首次适应算法等

  • 空闲链表法

  • 位示图法

  • 成组链接法：把顺序的 n 各空闲扇区地址保存在第一个空闲扇区内，其后一个空闲扇区则保存另一组顺序空闲扇区的地址，所有空闲扇区相互链接系统只需要保存一个指向第一个空闲扇区的指针

    超级块：放在卷头中，表示文件存储器空闲空间的位向量表或第一个成组链块，以及卷中目录区、文件区的划分信息

# 磁盘组织与管理

磁盘结构：柱面/磁道、盘面/磁头、扇区，最内道的扇区存储密度最大

磁盘读写延迟：

  • 寻道时间 ， 为启动磁臂的时间
  • 旋转延迟时间（定位到扇区） ， 为旋转速度（转/秒）—— 最主要的延迟
  • 传输时间 ， 为一个磁道上的字节数， 为读/写的字节数

🔺磁盘调度算法：

  • 先来先服务（FCFS）算法，平均寻道距离大
  • 最短寻找时间优先（SSTF）算法：会产生饥饿现象，不能保证全局时间最短
  • 扫描（SCAN）算法/电梯调度算法：走到底再回头，不利于远离磁头一端的访问请求，可避免饥饿现象
  • 循环扫描（C-SCAN）算法：磁头只单向移动来提供服务，走到底回返时直接回到起始端，途中不接受服务请求
  • LOOK 调度和 C-LOOK 调度算法：SCAN 和 C-SCAN 的优化形式，走到最后一个请求的磁道就返回，消除了对两端磁道请求的不公平

磁盘管理：

  • 磁盘初始化：低级格式化（物理分区），对物理分区进行逻辑格式化（创建文件系统），操作系统将初始的文件系统数据结构存储到磁盘上，包括空闲和已分配的空间及一个初始为空的目录

  • 拥有启动分区的磁盘称为启动磁盘或系统磁盘

  • 坏块：在 FAT（文件配置表）上标明/控制器维护一个磁盘坏块链表，控制器用备用块逻辑地代替坏块（扇区备用）

    坏块属于硬件故障，操作系统无法修复

引导控制块：引导操作系统，通常为分区的第一块

分区控制块：包括分区详细信息（超级块）

# I/O 管理

I/O 管理功能：状态跟踪、设备存取、设备分配、设备控制

I/O 设备分类

  • 按使用特性
    • 人机交互类外部设备
    • 存储设备
    • 网络通信设备
  • 按传输速率
    • 低速
    • 中速
    • 高速
  • 按信息交换的单位
    • 块设备：有结构，传输速率高，可寻址（磁盘）
    • 字符设备：无结构，传输速率低，不可寻址，采用中断驱动方式（打印机）

🔺I/O 控制方式

  • 程序直接控制方式：CPU 需要对外设状态进行循环检查

    优点：简单易实现

    缺点：CPU 和 I/O 设备只能串行工作，CPU 利用率低

  • 中断驱动方式：允许 I/O 设备主动打断 CPU 运行并请求服务

    由于数据中的每个字在存储器与 I/O 控制器之间的传输都必须经过 CPU，还是会消耗很多 CPU 的时间

  • DMA 方式：在 I/O 设备和内存之间开辟直接的数据交换通路

    特点：

    • 基本单位是数据块
    • 数据直接从设备进入内存
    • 仅在传送一个或多个数据块的开始和结束时，才需 CPU 干预，整块数据的传送是在 DMA 控制器的控制下完成的

    DMA 中设置 4 类寄存器：

    • 命令/状态寄存器（CR）
    • 内存地址寄存器（MAR）
    • 数据寄存器（DR）
    • 数据计数器（DC）

    工作方式：CPU 接收到 I/O 设备的 DMA 请求时，它给 I/O 控制器发出一条命令，启动 DMA 控制器，然后继续其他工作。之后 CPU 就把控制操作委托给 DMA 控制器，由该控制器负责处理。DMA 控制器直接与存储器交互，传送整个数据块，每次传送一个字，这个过程不需要 CPU 参与。传送完成后，DMA 控制器发送一个中断信号给处理器，因此只有在传送开始和结束时才需要 CPU 的参与。

  • 通道控制方式：把对一个数据块为单位的干预，减少为对一组数据块的干预，同时实现 CPU、通道和 I/O 设备三者并行操作

    只有一组数据读写的开始和结束需要 CPU 的参与。

    通道所执行的通道程序在内存中，与 CPU 共享内存。

    与 DMA 方式的区别：DMA 方式需要 CPU 来控制传输的数据块的大小、传输的内存位置，而通道方式中这些信息是由通道控制的。另外，每个 DMA 控制器对应一台设备与内存传递数据，而一个通道可以控制多台设备与内存的数据交换。

I/O 子系统的层次结构：

  • 用户层 I/O 软件

  • 设备独立性软件：向用户层提供统一接口

    设备独立性的优点：方便用户编程、使程序运行不受具体机器环境的限制、便于程序移植

  • 设备驱动程序

  • 中断处理程序

  • 硬件设备

I/O 核心子系统提供的服务：

  • I/O 调度：磁盘调度就是其中一种

  • 🔺缓冲与高速缓存

    • 磁盘高速缓存：利用内存中的存储空间来暂存从磁盘中读出的一系列盘块的信息，逻辑上属于磁盘，物理上是驻留在内存中的盘块（备份数据）

    • 缓冲区：位于内存

      目的：

      • 缓和 CPU 和 I/O 设备间速度不匹配的矛盾
      • 减少对 CPU 的中断频率，放宽对 CPU 中断响应时间的限制
      • 解决基本数据单元大小不匹配的问题
      • 提高 CPU 和 I/O 设备之间的并行性

      缓冲技术：I/O 设备输入到缓冲区时间为 ，缓冲区传送到工作区时间为 ，在工作区处理的时间为

      • 单缓冲，处理每块数据的时间为
      • 双缓冲，处理一块数据用时
      • 循环缓冲
      • 缓冲池

    区别：

    高速缓存存放的是备份，缓冲区不一定

    高速设备要访问的数据可能不在高速缓存中，因此直接访问低速设备；而高速设备和低速设备的通信都要经过缓冲区

  • 设备分配与回收

    设备分类：独占设备、共享设备、虚拟设备

    设备分配的数据结构：设备控制表（DCT）、控制器控制表（COCT）、通道控制表（CHCT）、系统设备表（SDT）、逻辑设备表（LUT）

    分配策略：

    • 静态分配（独占设备）：用户作业执行前一次性分配
    • 动态分配：在进程执行过程中根据需要分配

    分配算法：先请求先分配、优先级高者优先等

  • 假脱机（SPOOLing 技术）

    本质上是异步化 + 消息队列 + 完成回调，空间换时间

    输入缓冲区和输出缓冲区是在内存中开辟的两个缓冲区

    输入设备-->输入缓冲区-->输入井-->CPU 读入内存

    CPU-->输出井-->输出缓冲区-->输出设备

    特点：提高了 I/O 的速度；将独占设备改造为共享设备；实现了虚拟设备的功能

  • 设备保护

  • 差错处理
