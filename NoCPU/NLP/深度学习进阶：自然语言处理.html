# CH02 自然语言和单词的分布式表示

单词的分布式表示将单词表示为固定长度的向量。这种向量的特征在于它是用密集向量表示的。密集向量的意思是，向量的各个元素（大多数）是由非 0 实数表示的。

分布式假设：某个单词的含义由它周围的单词形成。（单词本身没有含义，单词含义由它所在的上下文（语境）形成）

窗口大小：上下文的大小（周围单词的数量）

基于统计的方法：在关注某个单词的情况下，对它的周围出现了多少次什么单词进行计数，然后再汇总

共现矩阵：汇总了所有单词的共现单词的矩阵

余弦相似度（cosine similarity）：设有 和 两个向量，则余弦相似度为： $$ similarity(x, y)=\frac{x\cdot y}{\Vert x\Vert \Vert y\Vert}= \frac{x_1y_1+\dots +x_ny_n}{\sqrt{x_1^2+\dots +x_n^2}\sqrt{y_1^2+\dots + y_n^2}} $$ 点互信息（Pointwise Mutual Information, PMI）： 表示 发生的概率， 表示 发生的概率， 表示 和 同时发生的概率，PMI 值越高，相关性越强。（考虑了单词单独出现的次数） $$ PMI(x,y)=\log_2 \frac{P(x,y)}{P(x)P(y)} $$ 使用共现矩阵重写： $$ PMI(x,y)=\log_2 \frac{\frac{C(x,y)}{N}}{\frac{C(x)}{N} \frac{C(y)}{N}}=\log_2 \frac{C(x,y)\cdot N}{C(x)C(y)} $$ 防止 ，使用正的点互信息（PPMI）： $$ PPMI(x,y)=max(0, PMI(x,y)) $$ PPMI 矩阵中很多元素都是 0，也就是说每个元素拥有的“重要性”很低。另外，这样的向量也容易收到噪声影响，稳健性差。对于这些问题，一个常见的方法是向量降维。

降维：从稀疏向量中找出重要的轴，用更少的维度对其进行重新表示。结果，稀疏矩阵就会被转化为大多数元素均不为 0 的密集矩阵。

奇异值分解（Singular Value Decomposition, SVD）：将任意矩阵分解为 3 个矩阵的乘积： $$ X=USV^T $$

U 是正交矩阵，这个正交矩阵构成了一些空间中的基轴（基向量），我们可以将矩阵 U 作为“单词空间”。S 是对角矩阵，奇异值在对角线上降序排列。简单地说，我们可以将奇异值视为“对应的基轴”的重要性。

矩阵 S 的奇异值小，对应的基轴的重要性低，因此，可以通过去除矩阵 U 中的多余的列向量来近似原始矩阵

# CH03 word2vec

基于计数的方法的问题：对大规模语料库进行处理时计算 SVD 复杂度太大

基于计数的方法使用整个语料库的统计数据（共现矩阵和 PPMI 等），通过一次处理（SVD 等）获得单词的分布式表示。而基于推理的方法使用神经网络，通常在 mini-batch 数据上进行学习。这意味着神经网络一次只需要看一部分学习数据（mini-batch），并反复更新权重。

推理：基于周围的单词（上下文），预测某处的单词是什么。基于推理的方法接受上下文信息作为输入，并输出（可能出现的）各个单词的出现概率。

CBOW 模型：简单的二层神经网络

  • 输入：上下文 one-hot

  • 输出：各个单词的概率

  • 对上下文考虑 N 个单词，那么输入层就有 N 个，此时中间层的神经元是 ，多个输入层共享权重

  • 从输入层到中间层的变换由全连接完成，此时全连接层的权重就是单词的分布式表示

  • 中间层的神经元数量比输入层少：中间层需要将预测单词所需的信息压缩保存，从而产生密集的向量表示。这时，中间层被写入了我们人类无法解读的代码，这相当于编码工作。而从中间层的信息获得期望结果的过程则称为解码。这一过程将被编码的信息复原为我们可以理解的形式。

    在行方向上保存了单词的分布式表示，而 在列方向上保存了单词的分布式表示（在 word2vec 中，只使用 作为最终的单词的分布式表示）

  • 从概率的角度

    当给定上下文 和 时目标词为 的概率（后验概率）： $$ P(w_t|w_{t-1}, w_{t+1}) $$ 套用交叉熵损失函数：，其中 是 one-hot 向量的元素，只有当这个 对应的 是正确解时 才为 1，因此可以得出： $$ L=-\log P(w_t|w_{t-1}, w_{t+1}) $$ 这也称为负对数似然，是一笔样本数据的损失函数，如果将其扩展到整个语料库，则损失函数可以写为： $$ L=-\frac{1}{T}\sum_{t=1}^T\log P(w_t|w_{t-1}, w_{t+1}) $$

skip-gram 模型：与 CBOW 模型要解决的问题相反

  • 输入：中间单词

  • 输出：上下文

  • 输入层只有一个，输出层的数量与上下文的单词个数相等，最后损失求和作为总损失

  • 从概率的角度，建模得出的概率表达式： $$ P(w_{t-1}, w_{t+1}|w_t) $$ 分解： $$ P(w_{t-1}, w_{t+1}|w_t)=P(w_{t-1}|w_t)P(w_{t+1}|w_t) $$ 带入交叉熵损失函数： $$ \begin{align*} L &= -\log P(w_{t-1}, w_{t+1}|w_t) \ &= -\log P(w_{t-1}|w_t)P(w_{t+1}|w_t) \ &= -(\log P(w_{t-1}|w_t)+\log P(w_{t+1}|w_t)) \end{align*} $$ 扩展到整个语料库： $$ L=-\frac{1}{T}\sum_{t=1}^T(\log P(w_{t-1}|w_t) + \log P(w_{t+1}|w_t)) $$ 它的损失函数需要求各个上下文单词的预测对应的损失的总和

从单词的分布式表示的准确度来看，在大多数情况下，skip-gram 的结果更好。特别是随着语料库规模的增大，在低频词和类推问题的性能方面，skip-gram 模型往往会有更好的表现。此外，就学习速度而言，CBOW 模型比 skip-gram 模型要快，这是因为 skip-gram 模型需要根据上下文数量计算相应个数的损失，计算成本变大。

skip-gram 模型需要通过一个单词预测其周围的单词，难度更大。可以说 skip-gram 模型要解决的是更难的问题，经过这个更难的问题的锻炼，skip-gram 模型能提供更好的单词的分布式表示。

对比基于计数的方法和基于推理的方法：

  • 场景：需要向词汇表添加新词并更新单词的分布式表示的场景

    基于计数的方法，需要从头开始计算，重新生成共现矩阵、进行 SVD 等

    基于推理的方法，允许参数的增量学习，可以将之前学习到的权重作为下一次学习的初始值，在不损失之前学习到的经验的情况下，高效地更新单词地分布式表示

  • 性质：

    基于计数的方法，主要是编码单词的相似性

    基于推理的方法，除了单词的相似性外，还能理解更复杂的单词之间的模式，可以解开类推问题

  • 准确度：差不多

# CH04 word2vec 的高速化

上面 CBOW 模型的不足：输入层 -> 中间层 -> 输出层 -> Softmax 层中的矩阵计算太耗时

解决办法：

  • one-hot 表示和权重矩阵的乘积：采用新的 Embedding 层来解决
  • Softmax 层的计算：引入 Negative Sampling 这一新的损失函数

这一乘积的作用实际上就是将权重矩阵的某一行抽取出来，所以没有必要使用矩阵乘法，可以创建一个从权重参数中抽取“单词 ID 对应行（向量）”的层，这里称为 Embedding 层。Embedding 来自词嵌入这一术语，也就是说，在这个 Embedding 层存放词嵌入（分布式表示）

单词的密集向量表示称为词嵌入或者单词的分布式表示

  • Embedding 层的正向传播只是从权重矩阵 W 中提取特定的行并传递给下一层

  • 在反向传播时，从上一层传过来的梯度将原样传给下一层，不过从上一层传来的梯度会应用到权重矩阵 dW 的特定行（idx）

  • Embedding 层保存单词的分布式表示，在正向传播时，提取单词 ID 对应的向量

    class Embedding:
    def __init__(self, W):
        self.params[W]
        self.grads = [np.zeros_like[W]]
        self.idx = None
        
    def forward(self, idx):
        # Embedding 层的正向传播只是从权重矩阵 W 中提取特定的行并传递给下一层
        W, = self.params
        self.idx = idx
        out = W[idx]
        return out

        # 上一层传来的梯度会应用到权重矩阵 dW 的特定行（idx）
        dW, = self.grads
        dW[...] = 0
        
        for i, word_id in enumerate(self.idx):
            dW[word_id] += dout[i]

        # dW[self.idx] = dout
        # 这种写法不对，因为当 idx 中元素出现重复时，梯度的保存就会发生覆盖
        return None

class EmbeddingDot:
    def __init__(self, W):
        self.embed = Embedding(W)
        self.params = self.embed.params
        self.grads = self.embed.grads
        self.cache = None
        
    def forward(self, h, idx):
        target_W = self.embed.forward(idx)
        out = np.sum(target_W * h, axis=1) # 对应元素乘积后逐行求和 shape: 行向量
        self.cache = (h, target_W)
        return out

前馈：神经网络的传播方向是单向的，信号仅在一个方向上传播。缺点：无法充分学习时序数据的性质（模式）

语言模型：给出了单词序列发生的概率。

考虑由 个单词 构成的句子，将单词按顺序出现的概率记为 （联合概率），使用后验概率分解： $$ P(w_1,\dots,w_m)=P(w_m|w_1,\dots,w_{m-1})P(w_{m-1}|w_1,\dots,w_{m-2})\dots P(w_3|w_1,w_2)P(w_2|w_1)P(w_1)\ =\prod_{t=1}^mP(w_t|w_1,\dots,w_{t-1}) $$ 由上述概率表示的语言模型也称为条件语言模型（conditional language model）

word2vec 是以获取单词的分布式表示为目的的方法，因此一般不会用于语言模型

RNN（Recurrent Nerual Network）循环神经网络具有一个机制，无论上下文有多长，都能将上下文信息记住。

RNN 的特征就在于拥有一个环路，这个环路可以使数据不断循环，通过数据的循环，RNN 一边记住过去的数据，一边更新到最新的数据。

各个时刻的 RNN 层接收传给该层的输入和前一个 RNN 层的输出，然后据此计算当前时刻的输出： $$ h_t=tanh(h_{t-1}W_h+x_tW_x+b) $$ 现在的输出 是由前一个输出 计算出来的，可以解释为，RNN 具有状态 h，并以上式的形式被更新。这就是说 RNN 层是具有状态的层或具有存储（记忆）的层的原因。

RNN 的 h 存储状态，事件每前进一步（一个单位），它就以上式的形式被更新，所以 RNN 的输出 被称为隐藏状态或隐藏状态向量。

将循环展开后的 RNN 可以使用常规的误差反向传播法。换句话说，可以通过先进行正向传播，再进行反向传播的方式求目标梯度。

因为这里的误差反向传播法是“按时间顺序展开的神经网络的误差反向传播法”，所以称为 Backpropagation Through Time（基于时间的反向传播），简称 BPTT。

要基于 BPTT 求梯度，必须在内存中保存各个时刻的 RNN 层的中间数据。因此，随着时序数据变长，计算机的内存使用量（不仅仅是计算量）也会增加，并且梯度逐渐变小，将无法向前一层传递。

在处理长时序数据时，通常的做法是将网络连接截成适当的长度。具体来说，就是将时间轴方向上过长的网络在合适的位置进行截断，从而创建多个小型网络，然后对截出来的小型网络执行误差反向传播法，这个方法称为 Truncated BPTT（截断的 BPTT）。

在 Truncated BPTT 中，网络连接被截断，但严格地讲，只是网络的反向传播的连接被截断，正向传播的连接依然被维持。也就是说，正向传播的信息没有中断地传播。与此相对，反向传播则被截断为适当的长度，以被截出的网络为单位进行学习。

在进行 RNN 的学习时，必须考虑到正向传播之间是有关联的，这意味着必须按顺序输入数据；反向传播则以块为单位，块内有顺序。

执行 mini-batch 的学习，需要让数据在各个批次中进行偏移。

例如，对长度为 1000 的时序数据，以时间长度 10 为单位进行截断，进行批次为 2 的学习，则作为 RNN 的输入数据，第一笔样本数据从头开始按顺序输入，第二笔数据从第 500 个数据开始按顺序输入（平移 500）。

  • 按顺序输入数据
  • 平移各批次（各样本）输入数据的开始位置

# CH06 Gated RNN

RNN 存在环路，可以记忆过去的信息，其结构非常简单，易于实现，但是许多情况下它都无法很好地学习到时序数据的长期依赖关系。

原因：BPTT 会发生梯度消失和梯度爆炸的问题。随着时间的回溯，梯度会在传播的中途变弱（甚至没有包含任何信息），则权重参数将不会被更新。

  • 反向传播的梯度流经 tanh、+、MatMul 运算

    • + 的反向传播将上游传来的梯度原样传给下游，梯度值不变

    • tanh 的值小于 1.0，并且随着 x 远离 0，它的值在变小。这意味着，当反向传播的梯度经过 tanh 节点时，它的值会越来越小。因此，如果经过 tanh 节点 T 次，则梯度也会减少 T 次。

      RNN 层的激活函数一般使用 tanh 函数，但是如果改为 ReLU 函数，则有希望抑制梯度消失的问题

    • MatMul 节点：每次反向传播通过矩阵乘积 计算梯度。根据时序数据的时间步长，将这个计算重复相应的次数。这里需要注意的是，每一次矩阵乘法计算都是用相同的权重 。

      如果 是标量，则问题很简单：当 大于 1 时，梯度呈指数级增加（梯度爆炸）；当 小于 1 时，梯度呈指数级减小（梯度消失）。

      如果 是矩阵，此时，矩阵的奇异值将成为指标。简单而言，矩阵的奇异值表示数据的离散程度。根据这个奇异值（更准确地说是多个奇异值中的最大值）是否大于 1，可以预测梯度大小的变化。

  • 梯度爆炸的对策：梯度裁剪 $$ if \ \ \ \Vert \hat g\Vert \ge threshold:\ \hat g=\frac{threshold}{\Vert \hat g\Vert}\hat g $$ 整合了神经网络中用到的所有参数的梯度，将阈值设置为 ，如果梯度的 L2 范数 大于或等于阈值，就按上述方法修正梯度

  • 梯度消失的对策：需要从根本上改变 RNN 层的结构（LSTM、GRU）

LSTM（长短期记忆——长时间维持短期记忆）：

LSTM 与 RNN 的接口的不同之处在于，LSTM 还有路径 c。这个 c 称为记忆单元，相当于 LSTM 专用的记忆部门，记忆单元 c 对外部不可见（不输出到下一层）。

LSTM 有记忆单元 ，这个 存储了时刻 t 时 LSTM 的记忆，可以认为其中保存了从过去到时刻 t 的所有必要信息（或者以此为目的进行了学习）。然后，基于这个充满必要信息的记忆，向外部的层（和下一时刻的 LSTM）输出隐藏状态 。

当前的记忆单元 是基于三个输入 经过某种计算算出来的，这里的重点是隐藏状态 要使用更新后的 来计算。另外，这个计算是 ，表示对 的各个元素应用 函数。

门控系统：开合程度为 0.0 ~ 1.0，该程度是从数据中自动学习到的。有专门的权重参数用于控制门的开合程度，这些权重参数通过学习被更新。另外 Sigmoid 函数用于求门的开合程度。

输出门：对 施加门。换句话说，针对 的各个元素，调整它们作为下一时刻的隐藏状态的重要程度。由于这个门管理下一个隐藏状态 的输出，所以称为输出门。

  • 输出门的开合程度根据输入 和上一个状态 求出： $$ o=\sigma(x_tW_x^{(o)}+h_{t-1}W_h^{(o)}+b^{(o)}) $$

  • 之后 可由 和 计算出来： $$ h_t=o \odot tanh(c_t) $$

    :阿达玛乘积，即矩阵对应元素相乘

tanh 的输出是 -1.0 ~ 1.0 的实数。我们可以认为这个数值表示某种被编码的信息的强弱程度。

sigmoid 的输出是 0.0 ~ 1.0 的实数，表示数据流出的比例。

因此，在大多数情况下，门使用 sigmoid 函数作为激活函数，而包含实质信息的数据则使用 tanh 函数作为激活函数。

遗忘门：在记忆单元 上添加一个忘记不必要记忆的门

  • 遗忘门的遗忘程度根据输入 和上一个状态 求出，也有专门的权重： $$ f=\sigma(x_tW_x^{(f)}+h_{t-1}W_h^{(f)}+b^{(f)}) $$

  • 之后 可由 和 计算出来： $$ c_t=f \odot c_{t-1} $$

新的记忆单元：遗忘门从上一时刻的记忆单元中删除了应该忘记的东西，但是这样一来，记忆单元只会忘记信息。为此我们添加新的 tanh 节点，向这个记忆单元添加一些应当记住的新信息。

  • 基于 tanh 节点计算出的结果被加到上一时刻的记忆单元 上，这样一来，新的信息就被添加到了记忆单元中。这个 tanh 节点的作用不是门，而是将新的信息添加到记忆单元中。因此，它不用 sigmoid 函数作为激活函数，而是使用 tanh 函数。

  • tanh 节点的计算根据输入 和上一个状态 求出： $$ g=tanh(x_tW_x^{(g)}+h_{t-1}W_h^{(g)}+b^{(g)}) $$

输入门：输入门判断新增信息 的各个元素的价值有多大。输入门不会不经考虑就添加新信息，而是会对要添加的信息进行取舍。

  • 输入门根据输入 和上一个状态 求出，也有专门的权重： $$ i=\sigma(x_tW_x^{(i)}+h_{t-1}W_h^{(i)}+b^{(i)}) $$

  • 将 和 对应元素的乘积添加到记忆单元中，形成新的记忆： $$ c_t=f\odot c_{t-1}+g\odot i $$

&新的记忆单元:c_t=f\odot c_{t-1}+g\odot i \
&输出门:o=\sigma(x_tW_x^{(o)}+h_{t-1}W_h^{(o)}+b^{(o)}) \

  • 因为 ，所以记忆单元的反向传播仅流过 + 和 × 节点。+ 节点将上游传来的梯度原样流出，所以梯度没有变化；× 节点的计算并不是矩阵乘积，而是对应元素的乘积，每次都会基于不同的门值进行对应元素的乘积（原先 RNN 中是使用相同的权重矩阵重复多次矩阵乘积），所以它不会发生梯度消失。
  • 遗忘门认为“应该忘记”的记忆单元的元素，其梯度会变小；而遗忘门认为“不能忘记”的元素，其梯度在向过去的方向流动时不会退化（完整传播）。所以应该长期记住的信息能在不发生梯度消失的情况下传播。

    抑制过拟合的方法：

    • 增加训练数据
    • 降低模型复杂度
    • 对模型复杂度给予惩罚的正则化（L2 正则化会对过大的权重进行惩罚）

    Dropout：在训练时随机忽略层的一部分神经元，也可以被视为一种正则化。Dropout 随机选择一部分神经元，然后忽略它们，停止向前传递信号。这种“随机忽视”是一种制约，可以提高神经网络的泛化能力。

    不要在 LSTM 的时序方向上插入 Dropout 层，因为当模型学习时，随着时间的推移，信息会渐渐丢失。也就是说，因 Dropout 产生的噪声会随时间成比例地积累，所以应在深度方向上插入 Dropout 层。

    变分 Dropout 也能用在时间方向上，它地机制是同一层的 Dropout 使用相同的 mask。

    通过同一层的 Dropout 共用 mask，mask 被固定。如此一来，信息的损失方式也被固定，所以可以避免常规 Dropout 发生的指数级信息损失。

seq2seq 模型：也称为 Encoder-Decoder 模型，是基于编码器和解码器进行时序数据的转换，在这些编码器和解码器内部可以使用 RNN。

编码器利用 RNN 将时序数据转换为隐藏状态 ，这里的 RNN 可以使用 LSTM 或者简单 RNN 或者 GRU 等；编码器输出的向量 是 LSTM 层的最后一个隐藏状态，其中编码了翻译输入文本所需的信息。这里的重点是，LSTM 的隐藏状态 是一个固定长度的向量。说到底，编码就是将任意长度的文本转换为一个固定长度的向量。

解码器如何处理这个编码好的向量，从而生成目标文本呢？通过文本生成的模型。

解码器的 LSTM 层会接收编码器输出的向量 ，这个唯一、微小的改变使得普通的语言模型进化为可以驾驭翻译的解码器。

Encoder 类的层结构：

在使用 RNN 进行文本生成时，学习时和生成时的数据输入方法不同。在学习时，因为已经知道正确解，所以可以整体地输入时序方向上的数据。相对地，在推理时，则只能输入第一个通知开始的分隔符（本次为 "_"）。然后，基于输出采样 1 个字符，并将这个采样出来的字符作为下一个输入，重复该过程。这次的问题是加法，因此本次选择生成确定性的答案，所以仅选择得分最高的字符，而不是基于 Softmax 函数的概率分布进行采样。

# CH08 Attention

seq2seq 存在的问题：Encoder 输出的是固定长度的向量。这意味着，无论输入语句的长度如何，都会被转换为长度相同的向量。

编码器的改进：

  • 编码器的输出的长度应该根据输入文本的长度相应地改变，可以使用全部时刻的隐藏状态

  • 各个时刻的隐藏状态中包含了大量当前时刻的输入单词的信息，编码器输出的 hs 矩阵就可以视为各个单词对应的向量集合。

    因为编码器是从左向右处理的，所以严格来说，刚才的“猫”向量中含有 “吾輩” “は” “猫” 这3个单词的信息。考虑整体的平衡性，最好均衡地含有单词“猫”周围的信息。在这种情况下，从两个方向处理时序数据的双向 RNN（或者双向 LSTM）比较有效。

解码器的改进：

  1. 在最简单的 seq2seq 中，是将 Encoder 最后的隐藏状态放入了 Decoder 最初的隐藏状态（即只提取了 hs 的最后一行）

    • 大脑在翻译时，可以认为是专注于某个单词（或者单词集合），随时对这个单词进行转换（猫=cat），同样，我们可以让 seq2seq 学习“输入和输出中哪些单词与哪些单词有关”

    • 我们的目标是找出与“翻译目标词”有对应关系的“翻译源词”的信息，然后利用这个信息进行翻译。也就是说，我们的目标是仅关注必要的信息，并根据该信息进行时序转换，这个机制称为 Attention。

      猫=cat 这样的表示单词对应关系的信息称为对齐

    • 改进后的编码器的层结构：

      这个“某种计算”接收 Decoder 各个时刻的 LSTM 层的隐藏状态和编码器的 hs，然后，从中选出必要的信息，并输出到 Affine 层。

      上面的网络所做的工作是提取单词对齐信息，就是从 hs 中选出与各个时刻解码器输出的单词有对应关系的单词向量（行）。比如 Decoder 输出 “I” 时，从 hs 中选出 “吾輩” 的对应向量。

    • 直接从 hs 中选择某个向量，这一操作无法微分。但可以另行计算表示各个单词重要度的权重。

      然后计算这个表示各个单词重要度的权重和 hs 中对应的单词向量的加权和，可以获得目标向量。

      上下文向量 c 中包含了当前时刻进行变换（翻译）所需的信息。更确切地说，模型要从数据中学习出这种能力。

        self.cache = None

        t = hs * ar

      class AttentionWeight:

        self.params, self.grads = [], []
        self.softmax = Softmax()
        self.cache = None
    
    def forward(self, hs, h):
        N, T, H = hs.shape
        hr = h.reshape(N, 1, H).repeat(T, axis=1)

        self.attention_weight_layer = AttentionWeight()
        self.weight_sum_layer = WeightSum()
        self.attention_weight = None

残差连接（skip connection）：加深层另一个重要技巧（其他的有 Dropout、权重共享等）。所谓残差连接，就是指跨层连接。此时，在残差连接的连接处，有两个输出被相加。请注意这个加法（确切地说，是对应元素的加法）非常重要。因为加法在反向传播时按原样传播梯度，所以残差连接中的梯度可以不受任何影响地传播到前一个层。这样一来，即便加深了层，梯度也能正常传播，而不会发生梯度消失（或者梯度爆炸），学习可以顺利进行。

RNN 层的反向传播：

  • 梯度消失：LSTM、GRU 等 Gated RNN
  • 梯度爆炸：梯度裁剪

深度方向上：

  • 梯度消失：残差连接

Attention 层的应用：

  • GNMT（Google Neural Machine Translation）谷歌神经机器翻译系统

    技术：LSTM 层的多层化、BiLSTM、skip connection 等

  • Transformer：RNN 在并行处理上存在问题，因为其需要基于上一个时刻的计算结果逐步进行计算，因此基本不可能在时间方向上并行计算 RNN。Transformer 不用 RNN，而用 Attention 进行处理。

    Self-Attention：这是以一个时序数据为对象的 Attention，旨在观察一个时序数据==内部==每个元素与其他元素的关系

    Time Attention 层的两个输入中输入的是不同的时序数据（hs_enc 和 hs_dec），而 Self-Attention 的两个输入中输入的是同一个时序数据（hs），这样可以求得一个时序数据内各个元素之间的对应关系。

    Transformer 中用 Attention 代替了 RNN，编码器和解码器都使用了 Self-Attention。Feed Forward 层表示前馈神经网络（在时间方向上独立的网络），是具有一个隐藏层、激活函数为 ReLU 的全连接神经网络。Nx 表示灰色背景包围的元素被堆叠了 N 次。

  • NTM（Neural Turing Machine，神经图灵机）：RNN 和 LSTM 能够使用内部状态来存储时序数据，但是它们的内部状态长度固定，能塞入其中的信息量有限。因此，可以考虑在 RNN 的外部配置存储装置（内存），适当地记录必要信息。

    在带 Attention 的 seq2seq 中，编码器对输入语句进行编码。然后，解码器通过 Attention 使用被编码的信息。这里需要注意的仍是Attention 的存在。基于 Attention，编码器和解码器实现了计算机中的“内存操作”。换句话说，这可以解释为，编码器将必要的信息写入内存，解码器从内存中读取必要的信息。

    可见计算机的内存操作可以通过神经网络复现。我们可以立刻想到一个方法：在 RNN 的外部配置一个存储信息的存储装置，并使用 Attention向这个存储装置读写必要的信息。实际上，这样的研究有好几个，NTM 就是其中比较有名的一个。

    这里需要注意的是图中间的一个被称为“控制器”的模块。这是处理信息的模块，我们假定它使用神经网络（或者 RNN）。从图中可以看出，数据 “0” 和 “1” 一个接一个地流入这个控制器，控制器对其进行处理并输出新的数据。

    这里重要的是，在这个控制器的外侧有一张“大纸”（内存）。基于这个内存，控制器获得了计算机（图灵机）的能力。具体来说，这个能力是指，在这张“大纸”上写入必要的信息、擦除不必要的信息，以及读取必要信息的能力。顺便说一下，因为图 8-40 的“大纸”是卷式的，所以各个节点可以在需要的地方读写数据。换句话说，就是可以移动到目标地点。

    像这样，NTM 在读写外部存储装置的同时处理时序数据。NTM 的有趣之处在于使用“可微分”的计算构建了这些内存操作。因此，它可以从数据中学习内存操作的顺序。

    计算机根据人编写的程序进行动作。与此相对，NTM 从数据中学习程序。也就是说，这意味着它可以从“算法的输入和输出”中学习“算法自身”（逻辑）。

    层结构：

    这里 LSTM 层是控制器，执行 NTM 的主要处理。Write Head 层接收 LSTM 层各个时刻的隐藏状态，将必要的信息写入内存。Read Head层从内存中读取重要信息，并传递给下一个时刻的 LSTM 层。

    为了模仿计算机的内存操作，NTM 的内存操作使用了两个 Attention，分别是“基于内容的 Attention ”和“基于位置的 Attention ”。基于内容的 Attention 和我们之前介绍的Attention一样，用于从内存中找到某个向量（查询向量）的相似向量。

    而基于位置的 Attention 用于从上一个时刻关注的内存地址（内存的各个位置的权重）前后移动。这里我们省略对其技术细节的探讨，具体可以通过一维卷积运算实现。基于内存位置的移动功能，可以再现“一边前进（一个内存地址）一边读取”这种计算机特有的活动。

    重申一下，在读取（或者写入）内存中某个地址上的数据时，我们需要先“选择”数据。这个选择操作自身是不能微分的，因此先使用Attention 选择所有地址上的数据，再利用权重表示每个数据的贡献，这样就能够利用可微分的计算替代选择这个操作。
