# 方法

  • 方法是关联到某个类型的函数，相当于为这种类型定义了一个独占的方法

    func (c Celsius) String() string { return fmt.Sprintf("%g°C", c) }
// 调用
var c Celsius = 0.0
c.String()

    函数名前有一个附加的变量，叫做方法的接收器（receiver）（调用一个方法 == 向一个对象发送消息）

    在其他语言中，this 和 self 作为接收器，在 Go 中，接收器的名字可以任意取

​ 方法和变量名同名的话，通过 . 来调用会有歧义。

  • Go 中可以给任意类型（包括数值、Slice、map、字符串等）定义方法（只要这个命名类型的底层类型不是指针或者 interface），而不是像其他语言中只能给类定义方法。

  • 一个类型的方法名必须唯一，不同类型的方法可以重名。方法的好处在于在外部调用时可以省略包名（直接 变量.方法）

  • 基于指针对象的方法：可以为指针定义方法

    func (p *Point) ScaleBy(factor float64) {
  p.X *= factor
  p.Y *= factor
}
// 该方法的名字是 (*Point).ScaleBy

    在现实的程序里，一般会约定如果Point这个类有一个指针作为接收器的方法，那么所有Point的方法都必须有一个指针接收器，即使是那些并不需要这个指针接收器的函数。

    只有类型（Point）和指向它的指针（*Point），可以作为接收器。如果类型名本来就是一个指针，就不允许出现在接收器中（避免歧义）：

    type P *int
func (P) f() { /* ... */ } // compile error
    1
    2

    • 如果接收器是普通变量，但其方法需要接收指针，也可以用 变量.方法 的形式去调用，但这种方法只限于变量，不能通过一个无法取到地址的接收器去调用指针方法

      p := Point{1, 2}
p.ScaleBy(2)
// 等价于
(&p).ScaleBy(2) // 编译器会隐式地插入 & 操作符

    • 如果接收器是指针，但其方法需要接收变量，也可以用 指针.方法 的形式去调用（因为可以通过这个地址来找到变量）

      pptr := &p
pptr.Distance(q)
// 等价于
(*pptr).Distance(q) // 编译器会隐式地插入 * 操作符

      2

    • 总结：

      1. 接收器的实参和形参一致，为 T 或 *T，直接调用
      2. 接收器实参为 T，形参为 *T，编译器会自动加 &（仅限于可以取到地址的变量）
      3. 接收器实参为 *T，形参为 T，编译器会自动加 *

    • 接收器为非指针类型时，每次调用（作为入参）都会产生一次拷贝；反之，拷贝的是指针，仍指向原地址，修改会对原对象生效

      如果命名类型 T 的所有方法都是用 T 类型自己来做接收器（而不是*T），那么拷贝这种类型的实例就是安全的；调用他的任何一个方法也就会产生一个值的拷贝。比如 time.Duration 的这个类型，在调用其方法时就会被全部拷贝一份，包括在作为参数传入函数的时候。但是如果一个方法使用指针作为接收器，你需要避免对其进行拷贝，因为这样可能会破坏掉该类型内部的不变性。比如你对 bytes.Buffer 对象进行了拷贝，那么可能会引起原始对象和拷贝对象只是别名而已，实际上它们指向的对象是一样的。紧接着对拷贝后的变量进行修改可能会有让你有意外的结果。

  • nil 指针也是一个合法的接收器类型，尤其当 nil 对于对象来说是合法的零值时

    type IntList struct {
  Value int
  Tail *IntList
}

func (list *IntList) Sum() int {
  if list == nil {
    return 0
  }
  return list.Value + list.Tail.Sum()
}

    可以把 ColoredPoint 类型当作接收器来调用 Point 里的方法（Point 类方法也被引入了 ColoredPoint）：

    red := color.RGBA{255, 0, 0, 255}
blue := color.RGBA{0, 0, 255, 255}
var p = ColoredPoint{Point{1, 1}, red}
var q = ColoredPoint{Point{5, 4}, blue}
fmt.Println(p.Distance(q.Point)) // 5
p.ScaleBy(2)
q.ScaleBy(2)
fmt.Println(p.Distance(q.Point)) // 10
// p.Distance(q) // compile error

// 内嵌字段会指导编译器去生成额外的包装方法来委托已经声明好的方法
func (p ColoredPoint) Distance(q Point) float64 {
    return p.Point.Distance(q)
}

func (p *ColoredPoint) ScaleBy(factor float64) {
    p.Point.ScaleBy(factor)
}

    嵌入结构也可能是命名类型的指针：

    type ColoredPoint struct {
    *Point
    Color color.RGBA
}

p := ColoredPoint{&Point{1, 1}, red}
q := ColoredPoint{&Point{5, 4}, blue}
fmt.Println(p.Distance(*q.Point)) // "5" *(q.Point)
q.Point = p.Point                 // p and q now share the same Point
p.ScaleBy(2)
fmt.Println(*p.Point, *q.Point) // "{2 2} {2 2}"

    当编译器解析一个选择器到方法时，比如p.ScaleBy，它会首先去找直接定义在这个类型里的ScaleBy方法，然后找被ColoredPoint的内嵌字段们引入的方法，然后去找Point和RGBA的内嵌字段引入的方法，然后一直递归向下找。如果选择器有二义性的话编译器会报错，比如你在同一级里有两个同名的方法。

    方法只能在命名类型或指向命名类型的指针上定义，但通过内嵌可以在匿名结构体中嵌入方法。

    // 不采用嵌入 struct
var (
	mu sync.Mutex
  mapping = make(map[string]string)
)

func Lookup(key string) string {
  mu.Lock()
  v := mapping[key]
  mu.Unlock()
  return v
}

// 采用嵌入 struct
var cache = struct {  // 匿名 struct
  sync.Mutex
  mapping map[string]string
}{
  mapping: make(map[string]string)
}

func Lookup(key string) string {
  cache.Lock()
  v := cache.Mapping[key]
  cache.Unlock()
  return v
}

  • 执行某个方法：p.Distance(q) 可以被分成两步

    1. 得到选择器 p.Distance，选择器会返回一个方法值：一个将方法（point.Distance）绑定到特定接收器变量（p）的函数
    2. 调用这个函数，并传入函数的参数（q）。调用时不需要指定接收器（因为第一步已经绑定）
    p := Point{1, 2}
q := Point{4, 6}

// step 1
distanceFromP := p.Distance
// step 2
fmt.Println(distanceFromP(q)) // 5

var origin Point
// step 2
fmt.Println(distanceFromP(origin))

// step 1
scaleP := p.ScaleBy
// step 2
scaleP(2)

    • 方法值的用处：当一个 API 需要一个函数作为其参数，且这个函数应该是一个绑定了对象的方法，此时可以将方法值传入：

      type Rocket struct { /* ... */ }
func (r *Rocket) Launch() { /* ... */ }
r := new(Rocket)
// 不使用方法值：需要写一个匿名函数
time.AfterFunc(10 * time.Second, func() { r.Launch() })
// 使用方法值
time.AfterFunc(10 * time.Second, r.Launch)

    • 方法表达式：当调用一个方法时，与调用一个普通的函数相比，必须要用选择器语法（p.Distance）来指定方法的接收器，但通过方法表达式，可以像调用常规函数那样使用方法。方法表达式一般为 T.f 或 (*T).f，会返回一个函数值，这种函数会将其第一个参数用作接收器，所以可以用调用常规函数的方式来调用方法

      p := Point{1, 2}
q := Point{4, 6}

distance := Point.Distance // 方法表达式
fmt.Println(distance(p, q)) // 5
fmt.Printf("%T\n", distance) // func(Point, Point) float64

// 这个 Distance 实际上是指定了 Point 对象为接收器的一个方法
// func (p Point) Distance(q Point)
// 但 Point.Distance 多一个参数，需要第一个额外参数作为接收器
// 本书中函数和方法的区别是指有没有接收器
// 而其他语言中是指有没有返回值

scale := (*Point).ScaleBy // 方法表达式
scale(&p, 2)
fmt.Println(p) // {2, 4}
fmt.Printf("%T\n", scale) // func(*Point, float64)

      方法表达式的用处：根据一个不确定的变量来决定调用同一类型的哪个函数时

      type Point struct { X, Y float64 }

func (p Point) Add(q Point) Point { return Point{p.X + q.X, p.Y + q.Y} }
func (p Point) Sub(q Point) Point { return Point{p.X - q.X, p.Y - q.Y} }

type Path []Point

func (path Path) TranslateBy(offset Point, add bool) {
  var op func(p, q) Point Point // 方法表达式的函数签名
  if add {
    op = Point.Add
  } else {
    op = Point.Sub
  }
  for i := range Path {
    path[i] = op(path[i], offset)
  }
}

  • 封装（信息隐藏）：Go 语言只能通过大写首字母控制可见性，因此如果想要封装一个对象，必须将其定义为一个 struct，同时最小的封装单元是 package（其他语言是类）

    • 优点： 1. 使调用方只需要关注少量的语句 2. 隐藏实现的细节，防止调用方依赖那些可能变化的具体实现 3. 阻止调用方对对象内部的值任意地修改
