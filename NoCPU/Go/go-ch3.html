# 基础数据类型

  • 数据类型：

    • 基础类型：数字、字符串、布尔型
    • 复合数据类型：数组、结构体
    • 引用类型：指针、切片、字典、函数、通道
    • 接口类型

  • 整型：int 8 int16 int 32 int int64 uint8 uint16 uint32 uint64 uint，Unicode 字符 rune 类型与 int32 等价，byte 类型和 uint8 类型等价，无符号整数类型 uintptr，没有确定的 bit 大小但足以容纳指针

    取模 % 的符号和被取模数的符号一致；除法会截断余数。

    两个相同的整数类型可以进行比较

    &^ 位清空运算，^ 在被用作为一元运算符是表示按位取反

    有符号数的右移会用符号位来填充

    fmt.Printf() 使用：

    • %o 八进制
    • %x 十六进制
    • %b 二进制
    • %c 字符面值
    • %q 带单引号的字符
    • %g 浮点数（紧凑形式）
    • %e 浮点数（带指数）
    • %f 浮点数
    • %T 类型信息
    • %v 值的默认格式
    • %#v 相应值的 go 语法表示

    %08b 八位二进制，不够左补 0

    %#[1]o 带 0 前缀的八进制，复用第一个操作数

  • 浮点数：float32 float64

  • 复数：complex64 complex128 分别对应 float32 float64

    var x complex128 = complex(1, 2) // 1 + 2i
fmt.Println(real(x)) // 1
fmt.Println(imag(x)) // 2

  • 字符串：是不可变的字节序列，文本字符串通常被解释为采用 UTF8 编码的 Unicode 码点（rune）序列。s[i] 返回第 i 个字节的字节值。len() 函数返回的是字节数量（不是 rune 字符数目）。

    第 i 个字节不一定是字符串的第 i 个字符，因为对于非 ASCII 字符的 UTF8 编码会要两个或多个字节

    一个 UTF-8 字符 = 1～4 byte

    s := "Hello, 世界"
fmt.Println(len(s)) // 13

    不能原地修改字符串内部数据：

    s[0] = 'L' // error

    不变性意味着两个字符串共享相同的底层数据是安全的，所以复制字符串代价很低；一个字符串 s 和对应的子字符串切片 s[7:] 的操作也可以安全地共享相同的内存，所以字符串切片代价也很低。

    • 可以用十六进制 \xhh 或八进制 \ooo 将 Unicode 码点写到字符串面值中

    • 逐步构建字符串如果用 string 会导致很多分配和复制，可以使用 bytes.Buffer

      func intsToString(values []int) string {
	var buf bytes.Buffer // 不用初始化，因为零值也是有效的
	buf.WriteByte('[')
	for i, v := range values {
		if i > 0 {
			buf.WriteString(", ")
		}
		fmt.Fprintf(&buf, "%d", v)
	}
	buf.WriteByte(']')
	return buf.String()
}

func main() {
	fmt.Println(intsToString([]int{1, 2, 3})) // [1, 2, 3]
}

  • 常量：常量表达式的值在编译期计算，而不是在运行期。常量间的所有算术运算、逻辑运算和比较运算都是常量，对常量的类型转换或以下函数调用都是返回常量结果：len cap real imag complex unsafe.Sizeof

    • 常量可以是构成类型的一部分

      const arrlen = 4
var a [arrlen]int

	a = 1
  b
  c = 2
  d

fmt.Println(a, b, c, d) // "1 1 2 2"

    • 无类型常量：编译器为这些没有明确类型的数字常量提供比基础类型更高精度的算术运算（至少 256 bit）

      六种未明确类型的常量类型：无类型的布尔型（true false）、无类型的整数（0）、无类型的字符（\u0000）、无类型的浮点数（0.0）、无类型的复数（0i）、无类型的字符串（"124"）

      通过延迟明确常量的具体类型，无类型的常量不仅可以提供更高的运算精度，而且可以直接用于更多的表达式而不需要显式的类型转换

      var x float32 = math.Pi
var y float64 = math.Pi
var z complex128 = math.Pi

      除法运算符会根据操作数的类型生成对应类型的结果：

      var f float64 = 212
fmt.Println((f - 32) * 5 / 9) // "100" float64
fmt.Println(5 / 9 * (f - 32)) // "0"; 5 / 9 is an untyped integer 0
fmt.Println(5.0 / 9.0 * (f - 32)) // "100" 5.0 / 9.0 is an untyped float
