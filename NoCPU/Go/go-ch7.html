# 接口

  • Go 中的接口是隐式实现的，没有必要给某个具体类型实现所有满足的接口。这样可以使得创建新的接口类型时不会影响原具体类型的定义、同时可以扩展第三方包中的类型。

  • 接口类型是一种抽象的类型。它不会暴露出它所代表的对象的内部值的结构和这个对象支持的基础操作的集合；它们只会表现出它们自己的方法。也就是说当你有看到一个接口类型的值时，你不知道它是什么，唯一知道的就是可以通过它的方法来做什么。

  • 接口是一个合约：

    // io.Writer
type Writer interface {
  Write(p []byte) (n int, err error)
}

// Fprintf
func Fprintf(w io.Writer, format string, arg ...interface{}) (int, error)

// Printf
func Printf(format string, args ...interface{}) (int, error) {
  return Fprintf(os.Stdout, format, args...)
}

// Sprintf
func Sprintf(format string, args ...interface{}) string {
  var buf bytes.Buffer
  Fprintf(&buf, format, args...)
  return buf.String()
}

// os.Stdout type: *os.File
// &buf type: *bytes.Buffer
// *os.File 和 *bytes.Buffer 都实现了 Write 函数

    通过 Fprintf 接口，Printf 和 Sprintf 函数的功能可以复用 Fprintf，而且只要满足 io.Writer 这个接口，Fprintf 中的第一个参数可以随意的替换。

    因为 fmt.Fprintf 函数没有对具体操作的值做任何假设，而是仅仅通过 io.Writer 接口的约定来保证行为，所以第一个参数可以安全地传入一个只需要满足 io.Writer 接口的任意具体类型的值。一个类型可以自由地被另一个满足相同接口的类型替换，被称作可替换性（LSP里氏替换）。这是一个面向对象的特征。

	return len(p), nil

	fmt.Println(c) // 5

  • 接口值：由两部分组成，一个具体的类型和那个类型的值。它们被称为接口的动态类型和动态值。对于像Go 语言这种静态类型的语言，类型是编译期的概念；因此一个类型不是一个值。一些提供每个类型信息的值被称为类型描述符，比如类型的名称和方法。在一个接口值中，类型部分代表与之相关类型的描述符。

    • 接口的零值 var w io.Writer 表示它的 type 和 value 都是 nil。一个接口值基于它的动态类型被描述为空或非空，调用空接口会产生 panic。

    • 接口赋值（隐式类型转换）：w = os.Stdout // w = io.Writer(os.Stdout)，此时接口的 type 为 *os.File，value 为 os.Stdout 的拷贝（一个具体的 os.File 指针）

      编译器把代码生成在类型描述符的方法 Write 上，然后间接调用那个地址，这个调用的接收者是一个接口动态值的拷贝（os.Stdout）

      w.Write([]byte("hello"))
// =>
os.Stdout.Write([]byte("hello"))

    • 一个接口值可以持有任意大的动态值。两个接口相等仅当它们都是 nil，或者他们的动态类型相同并且动态值也根据这个动态类型的 == 操作相等。因为接口值可以比较，所以可以作为 map 的键或者 switch 的操作数。但只有在非常确定它们的动态值是可以比较的情况下，才去比较接口值。

  Interface // sort.Interface

func (r reverse) Less(i, j int) bool { // 覆盖了 data 提供的 Less 方法
  return r.Interface.Less(j, i)
}

func Reverse(data Interface) Interface {
  return reverse{data}

      func (db database) ServeHTTP(w http.ResponseWriter, req *http.Request) {
	switch req.URL.Path {
	case "/list":
		for item, price := range db {
			fmt.Fprintf(w, "%s: %s\n", item, price)
		}
	case "/price":
		item := req.URL.Query().Get("item")
		price, ok := db[item]

			return

	default:
		msg := fmt.Sprintf("no such page: %s\n", req.URL)

	}
}

  • 类型断言：使用在接口值上，检查它操作对象（必须是接口类型）的动态类型是否和断言的类型匹配 x.(T)

    • 如果断言的类型 T 是一个具体类型，并且检查成功，那么断言结果是 x 的动态值，并且类型为 T(静态类型)

      即具体类型的类型断言从它的操作对象中获取具体的值，如果检查失败，会抛出 panic

      var w io.Writer
w = os.Stdout
f := w.(*os.File) // success: f == os.Stdout
c := w.(*bytes.Buffer) // panic

f = bytes.NewBufferString("123") // error: f 此时类型已经固定为 *os.File
      1

    • 如果断言的类型 T 是一个接口类型，则检查 x 的动态类型是否满足 T。如果满足，动态值不会被获取，断言的结果仍然是一个有相同动态类型和值部分的接口值，但是结果为类型 T(动态类型）。换句话说，对一个接口类型的类型断言改变了类型的表述方式，改变了可以获取的方法集合（通常更大），但是它保留了接口值内部的动态类型和值的部分。

      var w io.Writer

      func writeString(w io.Writer, s string) (n int, err error) {
  // 新接口
  type stringWriter interface {
    WriteString(string) (n int, err error)
  }
  if sw, ok := w.(stringWriter); ok {
    return sw.WriteString(s)
  }
  return w.Write([]byte(s))
}

  • 类型分支

    • 接口以两种不同的方式使用

      1. 一个接口的方法表达了实现这个接口的具体类型之间的相似性（共性），但是隐藏了代码的细节和这些具体类型本身的操作。（子类型多态）
      2. 利用一个接口值可以持有各种具体类型值的能力，将这个接口认为是这些类型的联合。通过类型断言动态地区分这些类型。（非参数多态）即关注的是各个具体类型的个性，将这种方式使用的接口描述为 discriminated unions（可辨识联合）

    • 类型分支：基于这个接口值的动态类型使一个多路分支有效（只能在 switch 语句块中使用）

      switch x.(type) {
case nil: // ...
case int, uint: // ...
case bool: // ...
case string: // ...
default: // ...
}

      func sqlQuote(x interface{}) string {
  switch x := x.(type) {
  case nil:
    return "NULL"
  case int, uint:
    return fmt.Sprintf("%d", x)
  case bool:
    if x {
      return "TRUE"
    }  
    return "FALSE"
  case string:
    return sqlQuoteString(x)
  default:
    panic(fmt.Sprintf("unexpected type %T: %v", x, x))
  }
}
