# 反射

  • 一种机制：能够在运行时更新变量和检查他们的值、调用他们的方法和他们支持的内在操作，而不需要在编译时就知道这些变量的具体类型。反射让我们将类型本身作为第一类的值类型处理。

  • 为何需要反射？可能在函数中需要处理一类不满足相关接口的值，但这种类型没有特定的表示方式，或者还不存在。（所以需要一种手段来检查位置类型）

  • reflect.Type：一个接口类型，表示一个 Go 类型。函数 reflect.TypeOf 接收任意的 interface{} 类型，并以 reflect.Type 形式返回其动态类型的接口值（具体的类型）

    t := reflect.TypeOf(3)
fmt.Println(t) // "int"
fmt.Println(t.String()) // "int"

var w io.Writer = os.Stdout
fmt.Println(reflect.TypeOf(w)) // *os.File

    将一个具体的值转为接口类型会有一个隐式的接口转换操作，它会创建一个包含两个信息的接口值：操作数的动态类型和它的动态值。

    reflect.Type 接口满足 fmt.Stringer 接口，并且 fmt.Printf("%T", x) 中内部也使用了 reflect.TypeOf() 来输出。

  • reflect.Value：可以装载任意类型的值。函数 reflect.ValueOf 接收任意的 interface{} 类型，并返回一个装载着其动态值的 reflect.Value，该函数返回的结果是具体类型的值。reflect.Value 也可以持有一个接口值。

    v := reflect.ValueOf(3)
fmt.Println(v) // "3"
fmt.Println(v.String()) // "<int Value>"

var w io.Writer
w = os.Stdout
v := reflect.ValueOf(w)
fmt.Println(v.String()) // <*os.File Value>

    v := reflect.ValueOf(3) // reflect.Value
x := v.Interface() // interface{}

fmt.Printf("%d\n", i) // 3

    空接口会隐藏值内部的表示方式和所有方法，只有知道其具体的动态类型，借助类型断言才能访问到空接口内部的值；但是 Value 有很多方法来检查其内容。

    通过 v.Kind() 方法返回其 Kind 类型（底层类型）：Bool、String 和所有数字类型的基础类型；Array 和 Struct 对应的聚合类型；Chan、Func、Ptr、Slice 和 Map 对应的引用类型；interface 类型；还有表示空值的 Invalid 类型。（空的 reflect.Value 的 kind 即为 Invalid。）

    func formatAtom(v reflect.Value) string {
	switch v.Kind() {
	case reflect.Invalid:
		return "invalid"
	case reflect.Int, reflect.Int8, reflect.Int16,
		reflect.Int32, reflect.Int64:
		return strconv.FormatInt(v.Int(), 10)
  // ...
  case reflect.String:
		return strconv.Quote(v.String())
	case reflect.Chan, reflect.Func, reflect.Ptr, reflect.Slice, reflect.Map:
		return v.Type().String() + " 0x" +
			strconv.FormatUint(uint64(v.Pointer()), 16)
	default: // reflect.Array reflect.Struct reflect.Interface
		return v.Type().String() + " value"
	}
}

  • 一个递归的值打印器：

    func Display(name string, x interface{}) {
	fmt.Printf("Display %s (%T):\n", name, x)
	display(name, reflect.ValueOf(x))
}

func display(path string, v reflect.Value) {
	switch v.Kind() {
	case reflect.Invalid:
		fmt.Printf("%s = invalid\n", path)
	case reflect.Slice, reflect.Array:
		for i := 0; i < v.Len(); i++ {
			display(fmt.Sprintf("%s[%d]", path, i), v.Index(i))
		}
	case reflect.Struct:
		for i := 0; i < v.NumField(); i++ {
			fieldPath := fmt.Sprintf("%s.%s", path, v.Type().Field(i).Name)
			display(fieldPath, v.Field(i))
		}
	case reflect.Map:
		for _, key := range v.MapKeys() {
			display(fmt.Sprintf("%s[%s]", path,
				formatAtom(key)), v.MapIndex(key))
		}
	case reflect.Ptr:
		if v.IsNil() {
			fmt.Printf("%s = nil\n", path)
		} else {
			display(fmt.Sprintf("(*%s)", path), v.Elem())
		}
	case reflect.Interface:
		if v.IsNil() {
			fmt.Printf("%s = nil\n", path)
		} else {
			fmt.Printf("%s.type = %s\n", path, v.Elem().Type()) // 接口的动态类型
			display(path+".value", v.Elem())                    // 接口的动态值
		}
	default:
		fmt.Printf("%s = %s\n", path, formatAtom(v))
	}
}

func main() {
  e, _ := eval.Parse("sqrt(A / pi)")
	Display("e", e)

    反射可以访问到结构体中未导出的成员。通过 v.Field v.Index v.Elem v.MapIndex... 等操作间接获得的 Value，可能代表任意底层类型的值，包括接口类型。（如果直接用 reflect.ValueOf()，得到的只会是具体类型的值）

    即例如 v.Field(1).Kind() -> reflect.Interface，但 reflect.ValueOf(x).Kind() 不可能为 reflect.Interface

  • 通过 reflect.Value 修改值：变量（左值）是一个可寻址的内存空间，里面存储了一个值，并且存储的值可以通过内存地址来更新。

    所有通过 reflect.ValueOf(x) 返回的 reflect.Value 都不可取地址，但是通过指针解引用的方式，可以取到变量的地址：

    x := 2
a := reflect.ValueOf(x) // a 不可取地址
b := reflect.ValueOf(&x) // b 不可取地址
c := reflect.ValueOf(&x).Elem() // c 可以取地址

    可以通过 reflect.Value 的 CanAddr 方法来判断其是否可以被取地址：

    fmt.Println(a.CanAddr()) // false
fmt.Println(b.CanAddr()) // false
fmt.Println(c.CanAddr()) // true
    1
    2
    3

    每当我们通过指针间接地获取的 reflect.Value 都是可取地址的，即使开始的是一个不可取地址的 Value。

    • slice 索引表达式 e[i] 将隐式地包含一个指针，它就是可取地址的，即使一开始的 e 表达式不支持取地址也没关系
    • reflect.ValueOf(e).Index(i) 对应的值也是可取地址的，即使 reflect.ValueOf(e) 不可取地址。

    从变量对应的可取地址的 reflect.Value 来访问变量的步骤：

    1. .Addr()，它返回一个 Value，里面保存了指向变量的指针

    2. .Interface()，返回一个 interface{}，里面包含指向变量的指针

    3. 如果知道变量的类型，可以将得到的 interface{} 类型通过类型断言转换为普通的类型指针，这样就可以通过普通指针来更新变量

      x := 2
d := reflect.ValueOf(&x).Elem() // d refers to x
px := d.Addr().Interface().(*int) // px := &x
*px = 3
fmt.Println(x) // 3

      不使用指针也可以通过可取地址的 reflect.Value 变量的 .Set() 方法来更新对应的值：

      d.Set(reflect.ValueOf(4))
fmt.Println(x) // 4

      Set 方法将在运行时执行和编译时进行类似的可赋值性约束的检查。以上代码，变量和值都是 int类型，但是如果变量是 int64 类型，那么程序将抛出一个 panic 异常，所以关键问题是要确保改类型的变量可以接受对应的值：

      d.Set(reflect.ValueOf(int64(5))) // panic
      1

      还有很多用于基本数据类型的 Set 方法：SetInt SetUint SetString SetFloat 等。这些 Set 方法尽可能完成任务，比如 SetInt，只要变量是某种类型的有符号整数甚至命名的类型（只要底层数据类型是有符号整数就行） ，而且变量类型值太大会自动截断，但不能对一个引用 interface{} 类型的 reflect.Value 调用对于基本数据类型的 Set 方法：

      x := 1
rx := reflect.ValueOf(&x).Elem()
rx.SetInt(2) // ok
rx.Set(reflect.ValueOf(3)) // ok
rx.SetString("hello") // panic
rx.Set(reflect.ValueOf("hello")) // panic

var y interface{}
ry := reflect.ValueOf(&y).Elem()
ry.SetInt(2) // panic
ry.Set(reflect.ValueOf(3)) // ok
ry.SetString("hello") // panic
ry.Set(reflect.ValueOf("hello")) // ok

    stdout := reflect.ValueOf(os.Stdout).Elem() // os.Stdout -> *os.File
fmt.Println(stdout.Type()) // os.File

    一个可取地址的 reflect.Value 会记录一个结构体成员是否是未导出成员，如果是的话则拒绝修改操作。使用 CanSet 方法可以检查对应的 reflect.Value 是否是可取地址并可被修改的

    fmt.Println(fd.CanAddr(), fd.CanSet()) // true false

  • 获取结构体字段的标签：reflect.Type 的 Field 方法将返回一个 reflect.StructField，里面含有每个成员的名字、类型和可选的成员标签等信息。其中成员标签信息对应 reflect.StructTag 类型的字符串，并且提供了Get 方法用于解析和根据特定 key 提取的子串。

    v := reflect.ValueOf(ptr).Elem()
fieldInfo := v.Type().Field(1)
tag := fieldInfo.Tag
name := tag.Get("http") // 获得结构体成员标签 `http:"..."` 中的 ... 字符串值

  • 显示一个类型的方法集

    func Print(x interface{}) {
	v := reflect.ValueOf(x)
	t := v.Type()
	fmt.Printf("type %s\n", t)

	for i := 0; i < t.NumMethod(); i++ {
		methType := v.Method(i).Type()
		fmt.Printf("func (%s) %s%s\n", t, t.Method(i).Name,
			strings.TrimPrefix(methType.String(), "func"))
    // methType.String() -> func(paramType...) returnType...
}

    reflect.Type 的 Method 方法将返回一个 reflect.Method 的实例，对应一个用于描述一个方法的名称和类型的结构体

    reflect.Value 的 Method 方法将返回一个 reflect.Value 以表示对应的值；使用 reflect.Value.Call 方法可以调用一个 Func 类型的 Value

    v := reflect.ValueOf(time.Hour)
ret := v.Method(0).Call([]reflect.Value{}) // func (time.Duration) Hours() float64
fmt.Println(ret[0]) // 1
fmt.Println(ret[0].String()) // <float64 Value>
